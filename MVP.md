## Minimum Viable Product (MVP)

**Project:** Haunted Roguelite Typing Game

**Core Concept**

A browser-based, roguelite typing game with a ghost/haunted horror theme. The player progresses through waves of enemies by typing code snippets to defeat them, earning randomized upgrades between waves that persist until the run ends. The game targets beginner-to-intermediate computer science students who want to build typing fluency with real code syntax in an engaging format.

**MVP Feature Set**

The following features represent the minimum the team commits to completing by end of quarter:

1. **Wave-based gameplay** — The player faces successive waves of ghost/horror-themed enemies. Each enemy is defeated by correctly typing a single line from a code function, with the lines revealed progressively as the wave advances. Together, the wave builds toward a complete function displayed on screen in real time.

2. **Roguelite progression** — At the end of each wave, the player is presented with a randomized selection of upgrades (e.g., slow fall speed, extra lives, shorter snippets). Upgrades carry through the run and shape the experience.

3. **Boss encounter** — At the end of each wave, the player faces a boss that requires typing the complete function in full (the same one assembled line by line during that wave). This makes the boss feel like a natural payoff to the preceding wave rather than an isolated challenge.

4. **Coding language selection** — Before starting a run, the player can choose a language (e.g., JavaScript, Python) so that all snippets in that run correspond to their chosen language.

5. **Ghost/haunted horror aesthetic** — Cohesive visual theme throughout: ghost enemies, haunted environment, horror-inspired UI and sound design.

6. **Basic educational mechanic** — All typed content consists of real, syntactically valid code. Each wave builds a complete function line by line, and the boss requires reproducing it in full, reinforcing both syntax familiarity and function structure through repetition.

7. **End-of-run stats screen** — Displays accuracy, words per minute, waves survived, and score.

## Post-MVP Goals

Once the MVP is stable, the team aims to explore:

- <u>**Structured learning path**</u> — A campaign mode where the player progressively "builds" a website by typing HTML, then CSS, then JavaScript across levels, culminating in a rendered front-end project. The existing wave difficulty loop maps naturally onto this structure: HTML waves serve as the early, easier stages, CSS as intermediate, and JavaScript as the most challenging, so the learning progression and the game progression reinforce each other.
- <u>**Difficulty / session length options**</u> — A pre-game menu letting players choose how long a run they want (how complicated of a front-end they want to build), making the game accessible to different audiences and time constraints.
- **Jump scares and advanced horror effects** — Glitch effects, panic mode on low health, and atmospheric escalation as waves progress.
- **Conceptual question obstacles** — Certain enemies trigger a multiple-choice code comprehension question, answered by typing the correct option, blending typing practice with conceptual learning.
- **Leaderboard / high score tracking** — Persistent scoring across sessions.

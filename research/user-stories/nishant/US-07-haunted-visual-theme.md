# US-07 · UX · Sprint 2

## Understanding the haunted visual theme

**As a player**, I want the game to look and feel like a haunted horror environment, so that the theme creates tension and makes the typing challenges more exciting.

---

## Acceptance criteria

- [ ] Ghost sprites or silhouettes are used as enemy visuals
- [ ] Background art conveys a haunted setting (graveyard, mansion, fog)
- [ ] UI elements (health bars, timer, code panel) use horror-appropriate styling
- [ ] Ambient sound effects and music play during waves

---

## Notes

Visual and audio polish is what makes the concept land for players. The aesthetic should lean into classic horror atmosphere (fog, candlelight, gothic typography) rather than gore. The code panel is a UI centrepiece — giving it a "spectral terminal" look (e.g., faint glow, aged paper texture, ghostly font) helps integrate the educational mechanic with the theme. Sound design: ambient wind/creaking for waves, a distinct sting for boss spawns, a satisfying sound on enemy defeat.

## Dependencies

- Enemy system must exist to attach sprites to (see US-01)
- Boss encounter needs distinct visual/audio treatment (see US-03)

## Definition of done

The game is visually cohesive with ghost enemy sprites, a haunted background, horror-styled UI, and ambient audio playing throughout a wave. A player shown a screenshot can identify the horror theme immediately.

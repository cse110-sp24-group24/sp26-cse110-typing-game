# US-10 · Replayability · Sprint 3

## Feeling progress — run summary after defeat or completion

**As a new CS student**, I want a short run summary after each game, so that I can see what I practiced, how well I performed, and what to improve next time.

---

## Acceptance criteria

- [ ] The game shows a summary screen when a run ends
- [ ] The summary includes total waves survived or progress reached
- [ ] The summary includes at least one typing-related stat, such as tokens completed or accuracy
- [ ] The summary includes a breakdown of HTML, CSS, and JavaScript tokens typed, if tracked
- [ ] The player can choose to replay or return to the main menu from the summary screen

---

## Notes

This gives closure and reinforces the educational side of the game. It helps players reflect on performance instead of just losing and immediately restarting.

## Dependencies

- Run state tracking
- End-of-run UI flow
- Stats collection during gameplay

## Definition of done

At the end of a run, the player sees a readable summary of performance and can quickly choose what to do next.

# US-04 · Enemy Identity · Sprint 2

## Recognizing enemy types — language-based enemy classes

**As a new CS student**, I want HTML, CSS, and JavaScript enemies to behave differently, so that the game feels varied and helps me remember the role of each language.

---

## Acceptance criteria

- [ ] The game includes at least three enemy classes: HTML, CSS, and JavaScript
- [ ] Each class has a distinct visual identity
- [ ] Each class has at least one distinct gameplay trait
- [ ] HTML, CSS, and JavaScript traits are explained or hinted at clearly enough for a new player to understand
- [ ] Class differences are noticeable during gameplay without being confusing

---

## Notes

A simple mapping works best: HTML enemies can feel structural or durable, CSS enemies can feel movement- or control-based, and JavaScript enemies can feel reactive or higher-threat. The traits should be readable, not overly complex.

## Dependencies

- Enemy rendering and state system
- Token category system
- Basic visual design language for enemy types

## Definition of done

Players can tell that different enemy classes exist, recognize them visually, and notice that they behave differently during combat.

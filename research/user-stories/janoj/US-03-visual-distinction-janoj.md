# US-11 · Accessibility · Sprint 3

## Staying focused — clear visual distinction between syntax categories

**As a new CS student**, I want HTML, CSS, and JavaScript enemies to look clearly different from one another, so that I can quickly recognize what kind of syntax I am typing during fast-paced gameplay.

---

## Acceptance criteria

- [ ] HTML, CSS, and JavaScript enemies each use a distinct and consistent visual style
- [ ] Category differences are recognizable without relying only on color
- [ ] Enemy labels remain readable during movement and combat
- [ ] Visual distinctions are used consistently in gameplay and any related HUD elements
- [ ] A new player can identify that multiple syntax categories exist after a short play session

---

## Notes

Since the player is learning while playing, enemy presentation should help reinforce the difference between HTML, CSS, and JavaScript instead of making all tokens feel the same. Using shape, icon, border style, or subtle symbols in addition to color will also make the game easier to read quickly and more accessible.

## Dependencies

- Enemy class system
- Enemy rendering and styling
- Token category mapping

## Definition of done

During gameplay, players can quickly tell whether an enemy belongs to HTML, CSS, or JavaScript based on its presentation, and this distinction remains readable even in more crowded waves.

# US-02 · Content Packs · Sprint 1

## Learning by category — syntax grouped by language

**As a new CS student**, I want syntax tokens to be grouped into HTML, CSS, and JavaScript categories, so that I can associate what I type with actual areas of web development.

---

## Acceptance criteria

- [ ] The game includes separate token pools for HTML, CSS, and JavaScript
- [ ] Each token is labeled internally with its language category
- [ ] The game can spawn enemies from a chosen category or a mixed set
- [ ] Tokens in the beginner set are short enough to be typed comfortably on desktop and mobile
- [ ] The content structure supports adding more tokens later without changing gameplay code

---

## Notes

This story supports both learning and expandability. Players should gradually recognize that different token types belong to different parts of frontend development.

## Dependencies

- Token/content data structure
- Enemy assignment system
- Early balancing for beginner-friendly token length

## Definition of done

The game can load and use HTML, CSS, and JavaScript token sets in a structured and expandable way.

# US-07 · Feedback · Sprint 2

## Knowing how I’m doing — clear typing feedback

**As a new CS student**, I want immediate visual feedback when I type correctly or incorrectly, so that I can quickly learn from mistakes and stay engaged during combat.

---

## Acceptance criteria

- [ ] Correct input visibly advances progress on the targeted enemy
- [ ] Incorrect input gives immediate feedback
- [ ] Feedback is visible without covering the play area too much
- [ ] Completing a token produces a satisfying success effect
- [ ] The feedback system works consistently for HTML, CSS, and JavaScript tokens

---

## Notes

Because the player is learning both gameplay and syntax, feedback is essential. It should be clear enough to help learning but not so dramatic that it becomes distracting.

## Dependencies

- Input handling
- Enemy targeting and progress display
- Basic visual/audio feedback system

## Definition of done

Players can tell right away when they are typing correctly, when they make a mistake, and when a token has been fully completed.

# US-06 · Mobile UX · Sprint 2

## Playing between classes — mobile-friendly controls

**As a new CS student**, I want the game to be easy to play on my phone, so that I can practice code syntax even when I do not have my laptop with me.

---

## Acceptance criteria

- [ ] The game layout adapts to common mobile screen sizes
- [ ] The input field is easy to locate, tap, and use on mobile
- [ ] Enemy labels remain readable on smaller screens
- [ ] Core gameplay remains functional in portrait or a clearly supported mobile orientation
- [ ] Mobile players are not required to type unusually long syntax strings in the beginner experience

---

## Notes

The project prompt explicitly requires mobile friendliness. This story should prioritize readability, tap target size, and realistic typing difficulty on a phone.

## Dependencies

- Responsive layout system
- Input field design
- Beginner token balancing

## Definition of done

A player can open the game on a phone, clearly understand the layout, and complete a basic run without major usability problems.

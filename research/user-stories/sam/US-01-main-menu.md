# US-01 · UX · Sprint 1

## Reaching the game from the start screen

**As a player**, I want a start screen that presents the game title and a clear "Play" button, so that I can immediately understand what I'm playing and jump into a run without confusion.

---

## Acceptance criteria

- [ ] Game title and a one-line tagline are visible on load
- [ ] A "Play" button routes to the language selection screen
- [ ] No login or account creation is required to start
- [ ] The start screen uses the haunted horror visual theme

---

## Notes

The start screen is the first impression. It should feel atmospheric — not a bland white page with a button. A dark background, ghost silhouette, and styled title card set the tone before the player has typed a single character. Keep it simple: the game sells itself once you're in a wave, so don't over-explain upfront.

## Dependencies

- Language selection screen must exist as the routing destination (see US-05 in Nishant's stories)
- Haunted visual theme must be established to style this screen (see US-07 in Nishant's stories)

## Definition of done

A player who opens the game in a browser sees a styled start screen with the game title, tagline, and a "Play" button that routes directly to language selection. No authentication or setup is required.

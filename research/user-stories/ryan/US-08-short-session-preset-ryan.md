# US-08 · Session · Sprint 2

## Time limited practices

**As a new CS student**, I want to choose a short session preset (for example a fixed number of waves or a 10–15 minute target) before starting, so that I can make measurable progress in a bounded time box instead of going for endless runs.

---

## Acceptance criteria

- [ ] At run start, the player can pick at least one “short” preset alongside a default/normal run
- [ ] End-of-run stats clearly label the preset used
- [ ] Short presets still award a coherent arc: at least one upgrade decision if the arc includes multiple waves 
- [ ] No hidden penalties to XP/score that punish students for choosing shorter sessions—only transparent scaling if scores are compared

---

## Notes

New students might not have time to do full-on runs that can may go for extended periods of time.  

## Dependencies

- Wave progression and run state that can end early with a clean summary when a wave cap or time target is hit
- Run start flow where the player picks language and run options before the first wave

## Definition of done

A player can start a short preset, finish naturally within the advertised bounds on median skill, and see stats that feel complete rather than truncated.

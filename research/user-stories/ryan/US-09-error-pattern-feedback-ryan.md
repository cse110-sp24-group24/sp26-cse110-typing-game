# US-09 · Stats · Sprint 2

## Learning what kinds of mistakes I repeat

**As a new CS student**, I want the end-of-run summary to group my mistakes into a few actionable categories so that I leave with a concrete study target instead of only an accuracy percentage.

---

## Acceptance criteria

- [ ] End screen includes a compact breakdown (counts or percentages) across agreed categories that are honest for code typing
- [ ] At least one category addresses symbol confusion common in intro courses (e.g., `=` vs `==`, `:` vs `;`, bracket direction) where detectable
- [ ] If run is very short, show a friendly “not enough data” state instead of misleading charts
- [ ] No personally identifiable data required; categories are derived locally from the run

---

## Notes

Feature will give student immediate feedback, which is extremely importnat for learning.

## Dependencies

- Per-keystroke comparison data through the run and an end-of-run stats surface to attach breakdowns to

## Definition of done

After a few runs of similar length, a student can accurately state their top mistake category and attempt a focused warmup session targeting that weakness.

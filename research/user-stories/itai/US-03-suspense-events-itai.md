# US-03 · Theme · Sprint 2

## Staying engaged through haunted suspense events

**As a player with a short attention span**, I want occasional haunted suspense events during gameplay, so that the game keeps me alert and engaged while I practice typing.

---

## Acceptance criteria

- [ ] Suspense events can include flickering lights, distant sounds, shadow movement, or brief scare moments
- [ ] Events never cover the active code target or prevent the player from reading what to type
- [ ] Events are timed so they create tension without causing unfair mistakes
- [ ] A setting exists to reduce or disable intense scare effects

---

## Notes

The haunted theme should help with engagement, especially for players who get bored or sleepy during repetitive practice. The scares should be atmospheric and surprising, not punishing. A flicker or audio sting during downtime is better than hiding code while the player is typing.

## Dependencies

- Haunted visual and audio theme must exist
- Game state must expose safe moments where suspense events can trigger without interfering with input
- Accessibility settings must support reduced intensity

## Definition of done

During gameplay, haunted suspense events occasionally occur, improve the atmosphere, and do not block typing targets or cause unavoidable errors. Players can lower the intensity if needed.

# US-02 · Education · Sprint 2

## Reviewing what typed commands mean

**As a beginner coder**, I want brief flashcard-style explanations for commands I type, so that I learn what the syntax means instead of only memorizing the keystrokes.

---

## Acceptance criteria

- [ ] A flashcard can show the command or syntax pattern just practiced
- [ ] Each card includes a one-sentence beginner-friendly explanation
- [ ] Cards are optional or quickly dismissible so experienced players are not slowed down
- [ ] Flashcards use examples from the selected language or front-end topic

---

## Notes

The goal is to connect finger practice to conceptual understanding. If a student types `document.querySelector`, the game should be able to explain that it finds the first matching element on the page. Explanations should be written in plain language and avoid assuming the player already knows advanced vocabulary.

## Dependencies

- Code snippets must be tagged with important commands or syntax patterns
- UI must support a small explanation panel that does not obscure the main code area

## Definition of done

After typing a tagged line or completing a wave, the player can view a short explanation of a relevant command and return to gameplay with no loss of progress.
